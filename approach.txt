reigen — Approach Notes for Addressing Known Limitations
=========================================================

This document sketches how I would approach each limitation listed in
limitations.md. It is a plan, not an implementation.


1. DTC Kit Import
-----------------

1.1 Header Parsing Bug (Major)
  Root cause: the vendor readers in src/import/{ancestry,myheritage,ftdna,
  livingdna}.rs treat only `#`-prefixed lines as comments/headers and fall
  straight into integer parsing on the first non-`#` row. When that row is
  the column header (`rsid\tchromosome\tposition...`), parsing "position"
  as u32 fails.

  Approach:
    - Centralize header handling in src/import/vendor.rs (or a new
      `header.rs`) as a single `skip_header` helper that is vendor-agnostic.
    - Detection rule: a line is a header/comment if it starts with `#`,
      OR it contains none of the expected tokens for a data row (e.g. the
      first field is non-numeric for the chromosome column, or equals a
      known header token like "rsid", "chromosome", "position", "genotype",
      case-insensitively).
    - Apply it once at the top of each vendor parser before the row loop,
      so the existing per-row parse path remains strict.
    - Add regression fixtures under tests/fixtures/ for each vendor with
      and without the `#` prefix; assert byte-identical `.snp`/`.geno`
      output across both variants.

1.2 Compressed File Support (.zip, .gz)
  Approach:
    - Introduce an input-opening helper (e.g. `open_kit(path) -> Box<dyn
      BufRead>`) in src/import/mod.rs that dispatches on extension /
      magic bytes:
        * `.gz`  → flate2::read::MultiGzDecoder
        * `.zip` → zip crate; require exactly one regular file entry,
          otherwise error with a clear message listing entries.
        * plain → File directly.
    - Detect by magic bytes (first 2–4 bytes) in addition to extension,
      so misnamed files still work.
    - Thread the boxed reader through detect.rs (vendor sniffing must
      operate on the decompressed stream; sniff after open, not before).
    - Keep memory flat by streaming; never decompress to a temp file.
    - Add Cargo features `gz` and `zip` (default-on) so the dependency
      surface can be trimmed for minimal builds.

1.3 PACKEDANCESTRYMAP Space Inefficiency (single sample)
  This is a format-imposed minimum record size (48 bytes/SNP), not a bug.
  Approach:
    - Keep PAM output correct but emit a warning when nsamples == 1 that
      recommends TGENO and quotes the expected size ratio.
    - Add an `import --format auto` policy: for a single sample, default
      to TGENO; users opting into PAM explicitly get it without warning
      noise after the first.
    - Document the 48-byte floor inline in src/geno (wherever the PAM
      writer lives) so the constraint is discoverable at the code site.

1.4 Single Sample Only
  Approach:
    - Add `reigen import --batch <dir>` (or accept multiple positional
      paths) that runs the existing single-kit pipeline per input and
      then invokes the merge pipeline in-process — no intermediate files
      on disk unless `--keep-intermediates` is passed.
    - Sample IDs default to the kit filename stem; `--id-from <csv>`
      allows overriding from a manifest (path,sample_id[,family_id]).
    - Parallelize the per-kit import stage with rayon; serialize the
      merge stage (which is already the bottleneck for large N).


2. Dataset Merging
------------------

2.1 Format Support (reject EIGENSTRAT / TGENO as inputs)
  Approach:
    - Instead of a hard rejection, auto-convert non-PAM inputs to a
      temporary PAM file under the OS temp dir, merge, then clean up.
      This matches user expectations without changing the merge core.
    - Expose `--no-auto-convert` for users who want the old strict
      behavior (useful in scripts that should fail fast).
    - For TGENO specifically, transposition is already implemented in
      src/transpose.rs — reuse it rather than round-tripping through
      text.

2.2 Silent Conflict Dropping (no .missnp)
  Approach:
    - Extend src/merge/stream.rs so the reconciler emits a structured
      reason per dropped SNP: { rsid, chrom, pos, a1_src, a2_src,
      a1_ref, a2_ref, reason }.
    - Write these to `<output>.missnp.tsv` (PLINK-compatible rsid-only
      `.missnp` also emitted for tooling compatibility).
    - Keep the aggregate count in the log, but add a one-line pointer
      to the missnp file.
    - Gate behind `--missnp` (default on); `--missnp=off` preserves
      current behavior for users who do not want the extra file.

2.3 Memory Usage for TGENO Output
  Current cost: ~20k × 1.2M genotypes materialized in RAM for transpose.
  Approach:
    - Tile the transpose: process the matrix in sample-blocks sized to
      a configurable RAM budget (`--tgeno-tile-mb`, default 512MB).
    - For each tile, read the SNP-major chunk, transpose in place, and
      append to the output `.tgeno` using pre-computed byte offsets so
      appends land in the right place (requires two passes if we want
      a single output file — first pass writes header + sparse layout,
      second pass fills tiles).
    - Alternative: write one `.tgeno` shard per tile and concatenate at
      the end via mmap (avoids the two-pass dance on filesystems that
      support sparse files).
    - Benchmark against the 20k × 1.2M case in benches/ before shipping;
      target peak RSS under 1GB.


3. General Limitations
----------------------

3.1 Indel Support
  `reigen` is a SNP toolkit; full indel support is out of scope (the
  packed binary formats assume 2 alleles × 1 char). Realistic approach:
    - Detect indel rows early in the .snp/.bim reader and choose a
      policy from `--indels={error|skip|warn}`; default `warn` (skip
      the row, count it, keep going), which matches how most downstream
      pipelines cope.
    - Record skipped indel rows in a sidecar `.skipped_indels.tsv` so
      users can audit what was dropped.

3.2 Strand Reconciliation Default (A/T, C/G)
  The current default is correct; the limitation is really discoverability.
  Approach:
    - Emit the count of dropped ambiguous SNPs in the summary log with
      the exact flag to override (`--allow-ambiguous`).
    - Add `--ambiguous-policy={drop|keep|keep-if-maf-lt=0.4}` — the
      last option is the conservative middle ground used by several
      population-genetics pipelines (keep only where MAF is low enough
      that the strand call is unambiguous from frequency).

3.3 No PED (Text) Support
  Keep as out-of-scope. The practical approach is a documented
  one-liner using PLINK to convert `.ped`/`.map` → `.bed`/`.bim`/`.fam`,
  which `reigen` already consumes. Add this to README.md rather than
  implementing a text PED reader.


Implementation Order
--------------------
1. 1.1 header bug — small, high user impact, blocks most vendors.
2. 2.2 missnp output — small, unblocks debugging of merge failures.
3. 1.2 compressed input — medium, very visible quality-of-life win.
4. 3.1 indel policy flag — small, prevents fatal errors on real data.
5. 2.1 auto-convert on merge — medium.
6. 1.4 batch import — medium.
7. 2.3 tiled TGENO transpose — largest; do last, behind a benchmark.
8. 1.3 PAM warning + 3.2 ambiguous-policy — polish.


Testing Strategy
----------------
- Golden-file tests for every vendor (with and without `#` header, with
  and without gzip/zip wrappers).
- Property test for merge: for randomly generated pairs of PAM datasets
  with known overlap, assert that (a) no reconcilable SNP is dropped,
  (b) every dropped SNP appears in the missnp file with a valid reason.
- Benchmark in benches/ for the TGENO transpose memory ceiling; fail
  CI if peak RSS exceeds the configured budget by >20%.
